# Phase 2: 核心体验优化 — 完成

## 为什么改

Phase 1 处理了展示层后，两个核心体验缺失：面板位置不记忆、设置页功能太少。

## 改了哪些文件

| 文件 | 改动 |
|------|------|
| `src/settings.ts` | 新增 `widgetPosX/Y`、`cdnUrl` 配置字段；设置页新增语言选择器、CDN URL 配置、云端状态展示、debug 分区 |
| `src/ui/floating-widget.ts` | 创建 widget 时从 settings 恢复位置；拖拽结束时保存位置到 settings |
| `src/services/cloud-manager.ts` | 新增 `setCdnUrl()` 方法，支持动态切换 CDN 源；URL rewrite 使用实例变量而非硬编码 |
| `src/main.ts` | 初始化 cloudManager 时从 settings 读取 CDN URL |
| `src/lang/index.ts` | 新增 13 个 settings 相关 i18n key |
| `src/lang/zh.json` | 新增 13 个中文翻译 |

## 特意没改什么

- 框架层未动
- 适配器模板未动  
- 不涉及词典格式
- 不涉及插件间通信协议
- 位置字段向后兼容（undefined → 居中）

## 风险

低。新增字段有默认值，旧用户 settings 无这些字段时会 fallback 到默认行为。
